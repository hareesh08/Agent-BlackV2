Multi-Agent Research Assistant Ecosystem using MCP and A2A
Overview
The objective of this assignment is to build a collaborative multi-agent research ecosystem that leverages Large Language Models (LLMs), Prompt Engineering, Model Context Protocol (MCP), Agent-to-Agent (A2A) communication, and modern Agentic AI frameworks.




Team Structure
Each team will consist of three members.

Each member will be responsible for building one specialized research agent.


Team MemberSpecialized AgentMember AComputer Vision Research AgentMember BNLP Research AgentMember CMachine Learning Research Agent

Model availability:

Use any of the following or similar models that you can find here if Gemini's request limits get exhausted.

https://build.nvidia.com/google/gemma-3n-e4b-it, https://build.nvidia.com/nvidia/nemotron-mini-4b-instruct

Project Goal
The goal is to create a research assistant ecosystem where:

? Every member develops a specialized agent
? Every member exposes tools through MCP
? Every member deploys their agent as an A2A-compatible service
? Every member creates their own Host Agent
? Every Host Agent can communicate with:
? The owner's specialized agent
          ? Both teammates' specialized agents This ensures that every participant learns both:
1. Agent Development
2. Agent Consumption and Orchestration




Expected Architecture
Each participant will build:

1. Specialized Research Agent
A domain expert agent responsible for solving research-related problems in its specialization. Examples:
Computer Vision Agent

Specializes in:

? Image Classification
? Object Detection
? Segmentation
? Vision Transformers
? Vision-Language Models
? Medical Imaging
? Video Analytics

NLP Agent

Specializes in:

? LLMs
? Retrieval Augmented Generation
? Prompt Engineering
? Information Extraction
? Text Classification
? Summarization
? Conversational AI

Machine Learning Agent

Specializes in:

? Classical Machine Learning
? Model Selection
? Feature Engineering
? Time Series
? Experiment Design
? Evaluation Strategies
? Hyperparameter Optimization



2. MCP Server
Each specialized agent must expose its capabilities through MCP tools. The MCP server should act as the tool layer for the agent.
The agent should invoke MCP tools whenever external information or specialized processing is required.



3. A2A Server
Each specialized agent must be deployed as a service that can be accessed by teammates. The deployed service will act as an A2A endpoint.
Other Host Agents should be able to:

? Discover capabilities
? Send requests

? Receive responses



4. Host Agent
Each participant must create their own Host Agent. This Host Agent acts as an orchestrator.
Responsibilities:

? Understand user requests
? Break down tasks
? Decide which specialized agents to invoke
? Communicate with remote agents
? Aggregate responses
? Generate final output

Every participant will therefore have:

? One Specialized Agent
? One MCP Server
? One A2A Endpoint
? One Host Agent
